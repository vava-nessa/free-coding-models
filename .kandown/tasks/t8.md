# Hide router prompts and simplify sticky table header

## Context

The main TUI table should show only the application/version row, column header, model rows, and full footer. The visible router promotion and footer daemon status should be hidden from normal users, while the Shift+R shortcut remains active as an unadvertised tester entry point.

## Subtasks

- [x] Remove the Search/Tier/Provider/Verdict/Health header row from the main table
  report: The main table now renders only the app/version row and column header before model rows.
- [x] Remove the Smart Router top upgrade banner from the main table
  report: Removed the table-level upgrade banner render path and its exported overlay helper.
- [x] Hide footer router status text while keeping Shift+R functional
  report: Removed visible router footer status, help entry, command palette router entries, and README TUI shortcut docs. Re-enabled Shift+R in the key handler as an unadvertised tester shortcut.
- [x] Update tests, README/changelog, and version
  report: Updated sticky header/footer regression tests, README router TUI docs, CHANGELOG.md, and package.json for 0.3.57.
- [x] Run local verification and real npm package verification
  report: Local `pnpm test` passed with 361 tests across 65 suites. `pnpm build:web` passed. `pnpm start` rendered the table in a PTY with the simplified sticky header, bottom-stuck footer, no visible router banner/footer hint, and Shift+R successfully opened the hidden router dashboard. npm published 0.3.57, global install succeeded, and the published binary responded to `free-coding-models --help`.

## Notes

- Do not delete router functionality. The user explicitly wants Shift+R to keep opening it as an easter egg.
- The release should include a complete single-version changelog suitable for GitHub release notes.
